Contribute

Phproject is maintained as an open source project for use by anyone around the world. If you find a bug or would like to see a new feature added, feel free to create an issue on Github or submit a pull request with new code.

In order to ease debugging, change the DEBUG value to 3 in config.ini. In addition to giving more detail on error pages, level 3 enables displaying a query log in the page footer that includes query times and shows which queries were cached.

Translations

Phproject is translated by our wonderful community on Crowdin. If you would like to help with translations, contribute either directly to our dictionary on Crowdin, or translate our demo site in-context at translate.phproject.org. If a language you want to translate to is not listed in our Crowdin project, please open an issue on GitHub and we'll add the language.

Code Standards

In order to keep things clean and consistent, we've set some simple standards on the Phproject source code. Opening braces are always on the same line as the statement that they correspond with. In PHP code, double quotes should be preferred, unless the string needs to contain double quotes, then single should be preferred rather than escaping the double quotes. CSS and Javascript code should use single quotes when possible.

{% highlight php %} exists("id")) { return $this->get("id"); } return null; } } ?> {% endhighlight %}

Be sure to keep indentation clean and consistent with the rest of the project as well, as it makes finding a particular block of code much easier. These standards should be used whenever contributing to the main project or when writing a plugin for Phproject.
